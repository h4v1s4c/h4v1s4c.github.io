Una red es un conjunto de dispositivos conectados entre sí para intercambiar información. En informática, una red puede estar formada por dos equipos o por millones de ellos.

Sin embargo, no todas las redes cumplen la misma función ni tienen el mismo alcance. Una de las divisiones más importantes es la que distingue entre **redes privadas (LAN)** y **redes públicas (WAN)**.

Entender esta diferencia es una base esencial para empezar a estudiar redes.

## Qué es una red privada (LAN)

Una **LAN** (*Local Area Network*) es una red de área local. Se trata de una red que conecta dispositivos dentro de un espacio limitado, como una vivienda, una oficina, un aula o un edificio.

Por ejemplo, dentro de una red LAN pueden conectarse:

- un ordenador
- un móvil
- una impresora
- una consola
- una televisión inteligente
- un servidor local

Todos esos dispositivos forman parte de la misma red local y pueden comunicarse entre sí.

## Por qué una LAN se considera privada

Una LAN se considera una red privada porque pertenece a un entorno interno y controlado. No está pensada para estar expuesta directamente a todo Internet, sino para permitir la comunicación entre dispositivos de un mismo espacio.

Esto significa que una LAN puede existir incluso aunque no haya conexión a Internet. Por ejemplo, un ordenador podría seguir enviando archivos a una impresora o accediendo a un servidor local aunque la conexión externa se cayese.

## Qué es una red pública (WAN)

Una **WAN** (*Wide Area Network*) es una red de área amplia. A diferencia de una LAN, no se limita a un espacio pequeño, sino que conecta redes a gran escala.

La WAN más conocida es **Internet**, que en realidad no es una única red, sino una enorme red formada por muchas redes más pequeñas conectadas entre sí.

Mientras que una LAN conecta dispositivos cercanos dentro de un entorno local, una WAN permite que distintas redes privadas se comuniquen entre sí a larga distancia.

## Diferencia principal entre LAN y WAN

La diferencia más importante entre ambas es su alcance y función:

- una **LAN** conecta dispositivos dentro de un entorno cercano
- una **WAN** conecta redes entre sí a gran escala

Dicho de forma simple:

- la LAN es tu red local
- la WAN es la red externa que conecta muchas redes locales

## El papel del router

El router es el dispositivo que une ambas partes.

Dentro de casa o de una oficina, los dispositivos están conectados a la **LAN**. Cuando uno de esos dispositivos necesita acceder a una página web, enviar datos a un servidor o comunicarse con un servicio externo, el tráfico sale de la red local y pasa a través del router.

Por eso, el router actúa como puente entre la **red privada (LAN)** y la **red pública (WAN)**.

## Un ejemplo sencillo

Imagina una casa con varios dispositivos conectados al mismo router:

- un PC
- un móvil
- una Smart TV
- una consola

Todos esos dispositivos forman parte de la misma **LAN**.

Si el PC comparte un archivo con otro dispositivo dentro de casa, esa comunicación ocurre dentro de la red local.

Pero si el móvil abre una página web o la consola se conecta a un servicio online, entonces el tráfico sale de la LAN, pasa por el router y entra en la **WAN**, es decir, en Internet.

## LAN y Wi-Fi no son exactamente lo mismo

A veces se confunde la idea de LAN con el Wi-Fi, pero no son lo mismo.

El **Wi-Fi** es solo una forma de conectarse a la red local de manera inalámbrica. Una LAN puede funcionar tanto por cable como por Wi-Fi.

Por tanto, el Wi-Fi no es la red en sí, sino un medio de acceso a la red local.

## Resumen

Una **LAN** es una red privada que conecta dispositivos dentro de un entorno limitado, como una casa o una oficina.

Una **WAN** es una red de área amplia que conecta redes entre sí a gran escala. El ejemplo más claro es Internet.

El **router** es el dispositivo que conecta la red local con la red externa y permite el paso del tráfico entre ambas.

Distinguir entre LAN y WAN es uno de los primeros pasos para entender cómo se comunican los dispositivos dentro y fuera de una red.
